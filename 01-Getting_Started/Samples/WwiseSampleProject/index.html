Please start by installing the correct version of Wwise (2019.2.5 Build 7349) on your machine.

2019.2.5 Build 7349 can be downloaded and installed from the Wwise launcher: https://www.audiokinetic.com/download/.

In the Wwise Launcher, also check the Wwise SoundSeed Air and Wwise Convolution boxes. Both plug-ins are used in Microsoft Flight Simulator.

Once Microsoft Flight Simulator Wwise version and the SDK are installed, you can find the Wwise Project in your SDK package folder: MSFS SDK.

IMPORTANT: You can’t create a Wwise Project from scratch and use it to generate audio content for Microsoft Flight Simulator. Even if you create the same busses in the Master-mixer Bus hierarchy, they won’t be recognized by the game. To create new content, use only the shared WwiseSampleProject.

1/ Before beginning the implementation process

What you can do with the Wwise project

  • You can create your own Soundbanks and Packages,
  • You can to edit the Actor-Mixer Hierarchy and the Event Hierarchy, in order to create and integrate your own audio content,
  • You can create your own SimVar Game Parameters if the ones that are shared do not include the ones you need,
  • You can create your own Attenuation Shareset,
  • You can create your own Effects Shareset and use it in your Actor-Mixer Hierarchy (please keep in mind that moderate use of these effects is recommended, as they may have a significant impact on game performance)

What you can’t do with the wwise project

The base structure of the project is included in the init soundbank of Microsoft Flight Simulator. This structre will be maintained in time and it won’t be subject to any changes. It includes:

  • Master-Mixer-Hierarchy: you can use it to route your sounds, but you can’t make any edits on it (creating new output bus or auxiliary bus, adding some effects, etc). Please visit the Master Mixer Busses page in order to find out how to route your sounds.
  • States and Switches: you are able to use all shared States and Switches game syncs, but you cannot edit them or create new ones. Please visit the Shared states and switches page in order to find out how to use the shared game_sync.

Common mistakes to avoid

  • Mixing too low or too high: Microsoft Flight Simulator is mixed in compliance with the ITU-R BS 1770-3 recommandations. Every audio content providers should follow these guidelines, so together, we will ensure a consistent mix of all community audio content. Please visit the Mix your Aircraft page in order to find out how to deal with your aircraft loudness mix in Microsoft Flight Simulator.
  • Incorrect routing: please make sure to send your sounds to the right bus, so that the in-game mixer will apply to your aircraft. The Master Mixer Busses page will help you find out how to route your sounds.
  • Do not exceed the clip distance in your attenuation shareset:
    • Aircraft player: clipping distance is set up to 10,000 meters,
    • AI Aircraft: clipping distance is set up to 5,000 meters.

We invite you to explore the WwiseSampleProject and the aircraft samples so you can have an overview and a better understanding of the implementation process.

2/ Start by creating your SimObject workflow and packages

Create your SimObject Work Units

IMPORTANT: Because this SDK is live, updates are to be expected regularly on the shared content. In order to avoid your work to be overwritten by SDK future updates, your will need to build your working space by creating several WorkUnits in the WwiseSampleProject.

  • Start by creating your Actor-Mixer workunit, by going to the “Audio” tab in the Project explorer window. Name it witch your company name and followed by your aircraft name. For an AI aircraft, just add an "_AI" suffix.

  • Create your Event workunit by going to the “Event” tab, and also name it like your previous actor-mixer workunit. Same for AI aircraft, just add "_AI" suffix.

  • You’ll need to create your own workunit for all the content you’ll create:

    • ShareSets,
    • Game Paramters,
    • Sessions,
    • Queries.

Create Player and AI soundbanks

In the Wwise Project Explorer window go to the Soundbanks tab, and create your workunit and own aircraft soundbank. The name of the sounbank must be the one described in the sound.xml in the Wwisepackage element. For more information about the WwisePackage element, please see the sound.xml categories page. We recommend that you also name it just like you Actor-MIxer & Event Workunit : YourCompanyName_YourAircraftName.

And the same for an AI aircraft, just add "_AI" suffix to your ai soundbank.

Once the creation is completed, simply drag & drop your event wwu freshly created into your soundbank from the Wwise Soundbank layout.

NOTE: Generate your soundbanks using the GenerateSoundBank.bat file. This file is located in the wwise project root : SDK.bat

Create your Simobject package (PC.PCK)

You now have everything required to create your SimObject.PC.PCK file. To do so:

  • Launch the FilePackager.App.exe from your wwise installation path (WWise).
  • Open the PCK_PC.wfpproj located in the Wwise Project directory.
  • Create you own package, naming it as your Soundbank and adding the “PC.PCK” extension: Your AI packages also require the "_AI"suffix.
  • Simply Drag & Drop your Soundbank into your packages.
  • Generate
  • Copy & Paster the Package from the GeneratedSoundbanks folder in the WwiseSampleProject into the sound (or soundai) folder of your SimObject.

NOTE: A version of Microsoft Flight Simulator higher than 1.3 is required to load the soundbank (not publicly released yet).

Your PC.PCK file must now be referenced in your sound.xml to be loaded in Microsoft Flight Simulator:

  <WwisePackages>
          <MainPackage Name="YourCompanyName_YourAircraftName"/>
        </WwisePackages>

Your soundai.xml must also reference its package :

  <WwisePackages>
          <MainPackage Name="YourCompanyName_YourAircraftName_AI"/>
        </WwisePackages>

For more information about the WwisePackage xml element, please see the sound.xml categories page

Your soundbank and audio package are now ready to be loaded in Microsoft Flight Simulator. You will need to generate your soundbanks and packages using the GenerateSoundBank.bat file, each time you modify or create new content in Wwise.

The last step, to listen and test your audio content in the game, is to load your package at runtime using the build button in the Microsoft Flight Simulator Project_Editor. Open your project using the devmod toolbar. Select your aircraft, and clic on the Build package button:

Please refer to the Project Editor page for more information. ___

3/ Event Naming Convention

The wwise event naming convention is simple, but must be respected otherwise the sounds will not properly play. Every wwise event using the wwisedata=true attribute must be named following this convention: Play_SoundbankName_WwiseEvent.

  • SoundbankName: the name of your Soundbank or your PC.PCK file.
  • WwiseEvent: the name value that is set up in the sound.xml for your audio event in the WwiseEvent attribute. (if you haven’t already done so, go and see sound.xml introduction page)

For instance, with a Soundbank and PC.PCK file called “MyAircraft” :

  <WindSounds>
          <Sound WwiseEvent="wind_sound" WwiseData="true" />
        </WindSounds>

The resulting event name in Wwise for “wind_sound” would be: “Play_MyAircraft_wind_sound”

AI aircrafts events must be named by adding the "_AI_" suffix between SoundbankName name and WwiseEvent. The resulting event name for that AI sound in Wwise would be: “Play_MyAircraft_ai_wind_sound”

IMPORTANT NOTE: The naming rules described above apply to all sounds except EngineSoundPresets which have their own specificity, allowing to ensure backward compatibility with legacy systems. This convention is described below

Naming convention for EngineSoundPresets

Below is the naming convention for EngineSoundPresets:

Sound.xml WwiseEvent Event Name in Wwise SoundbankName=MyAircraft
EngineIndex=1
JetWhine Play_SoundbankName_engEngineIndex_jet_whine Play_MyAircraft_eng1_jet_whine
NonCombustion Play_SoundbankName_engEngineIndex_non_combustion Play_MyAircraft_eng1_non_combustion
Starter Play_SoundbankName_engEngineIndex_starter Play_MyAircraft_eng1_starter
CombustionStart Play_SoundbankName_engEngineIndex_combustion_start Play_MyAircraft_eng1_combustion_start
Combustion Play_SoundbankName_engEngineIndex_combustion Play_MyAircraft_eng1_combustion
Propeller Play_SoundbankName_engEngineIndex_prop Play_MyAircraft_eng1_prop
Rotor Play_SoundbankName_engEngineIndex_rotor Play_MyAircraft_eng1_rotor
Shutdown Play_SoundbankName_engEngineIndex_shutdown Play_MyAircraft_eng1_shutdown

For AI aircraft:

Sound.xml WwiseEvent Event Name in Wwise SoundbankName=MyAircraft
EngineIndex=1
JetWhine Play_SoundbankName_ai_engEngineIndex_jet_whine Play_MyAircraft_ai_eng1_jet_whine
NonCombustion Play_SoundbankName_ai_engEngineIndex_non_combustion Play_MyAircraft_ai_eng1_non_combustion
Starter Play_SoundbankName_ai_engEngineIndex_starter Play_MyAircraft_ai_eng1_starter
CombustionStart Play_SoundbankName_ai_engEngineIndex_combustion_start Play_MyAircraft_eng1_combustion_start
Combustion Play_SoundbankName_ai_engEngineIndex_combustion Play_MyAircraft_ai_eng1_combustion
Propeller Play_SoundbankName_ai_engEngineIndex_prop Play_MyAircraft_eng1_prop
Rotor Play_SoundbankName_ai_engEngineIndex_rotor Play_MyAircraft_ai_eng1_rotor
Shutdown Play_SoundbankName_ai_engEngineIndex_shutdown Play_MyAircraft_ai_eng1_shutdown

4/ Notes

Positioning your sounds in the 3d space

  • The EngineIndex that you set up in the sound.xml positions the engine sound to the coordinates of the corresponding engine(defined in the engine.cfg file). There is thus no need to set up a NodeName on engine sounds.
  • For AnimationSounds, the sound automatically play on the animation node, so there is no need for a NodeName to be set up either.
  • For other sounds, if no NodeName is set up through sound.xml, the sound will play at the center of the plane. But it’s always better to set up a NodeName so your sounds will play in the right place. See the sound.xml categories for more informations about Sound attributes in sound.xml.
  • Pay attention to the ConeHeading attribute in sound.xml especially for engine sound if you want to emphase its perspectives. By default, the engine node is facing forward. So if you want the exhaust sounds to be facing backwards, you’ll need to set up the cone heading using ConeHeading=180 (sound.xml categories) and attenuate the outer angle in wwise:
  • You can also set up a position for your sounds using the Wwise Emitter with Automation feature. It can be really usefull for cockpit loops, like buzz, fans..etc.

Handle the SimVars:

  • The SimVar Watcher is a very useful tool, and can be found in the SDK tool kit. As the name suggests, it allows to watch simvar values in real-time so it’s easier to setup your game parameter and your RTPC.
  • The Wwise Profiler can be a great ally to track errors and mistakes in the implementation process.
