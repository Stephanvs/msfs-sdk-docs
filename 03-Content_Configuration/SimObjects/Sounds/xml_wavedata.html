Wave Data Sounds Properties

  • WwiseEvents available
    • Legacy Mapping Table

The Wave Data approach using sound.xml files benefits from all the sound categories and common sounds element attributes. If you haven’t already done so, please check out sound_categories and xml_definition pages to see all the possibilities of sound.xml.

This approach does not require the wwise editor, and is therefore limited to the commons elements and attributes of the sound.xml. You can’t use wwisedata attributes like WwiseRtpc that requires the use of the wwise editor.

Again, with this approach your Sound elements in sound.xml must have the attibute Wwisedata="False" and a FileName attribute.

FORMATS: The sound engine can load PCM .wav in 16 bits per sample and Microsoft ADPCM .wav compressed format.

Legacy Mapping Table

If you are used to the sound.cfg system, this table will help you to understand the sound.xml attributes from the legacy sound system standpoint.

LEGACY
SOUND.CFG
PROPERTY		 Microsoft Flight Simulator
SOUND.CFG
PROPERTY		 Microsoft Flight Simulator
SOUND.XML
ATTRIBUTE		 XML Description
filename supported FileName Specifies the name of the .wav file to play. The .wav extension should not be specified. If FileName attribute have comma separated filenames (e.g. FileName=“encrash1,encrash2”),the simulator code will randomly choose to play one of the listed .wav files.
In sound.xml:FileName="Mysound01,Mysound02,Mysound03"
flags unsupported unsupported Flags are no longer supported in Microsoft Flight Simulator. You can use SimVars Sounds element to trigger sounds according to the conditions you want.
viewpoint supported ViewPoint Determines whether the sound plays in the cockpit or outside or both. Without these parameters the sound will play in both viewpoints.
In sound.xml :
ViewPoint="Inside"
vparams supported VolumeCurve Defines the sounds amplitude envelope. The units for volume are the same as in FSX. They are linear, with a value of 50 meaning -3dB attenuation, and 0 meaning silence. The main difference with the legacy SOUND.CFG system is that you can connect the volume curve to the SimVars of your choice. Here is an example of the xml structure :<VolumeCurve SimVar="GENERAL ENG PCT MAX RPM" Units="PERCENT OVER 100" Index="1">
<Point SimValue="0.005" Output="0.0" />
<Point SimValue="0.127" Output="3.2" />
<Point SimValue="0.197" Output="16.8" />
<Point SimValue="0.238" Output="76.8" />
<Point SimValue="0.336" Output="78.4" />
<Point SimValue="0.405" Output="18.4" />
<Point SimValue="0.590" Output="0.0" />
<Point SimValue="0.630" Output="0.0" />
</VolumeCurve>
rparams supported PitchCurve Defines the sounds pitch envelope. Represents the sound’s relative pitch (and, invariably, the playback speed) as a function of a generic value. The format and behavior of rparams is similar to vparams, except that the second value of each point represents a pitch scaler. A value of 1.0 specifies that the sound file is played at unity pitch. A value of 2.0 specifies that the file is played an octave higher and twice as fast. The main difference with the legacy SOUND.CFG system is that you can connect the pitch curve to the SimVars of your choice. Here is an example of the xml structure :<PitchCurve Index="1" SimVar="GENERAL ENG PCT MAX RPM" Units="PERCENT OVER 100">
<Point SimValue="0.266" Output="0.955" />
<Point SimValue="0.556" Output="1.131"/>
</PitchCurve>
minimum_volume supported unsupported This parameter is no longer supported in sound.xml. But you can use VolumeCurve & PitchCurve on the sound you want.
maximum_volume supported unsupported This parameter is no longer supported in sound.xml. You can use VolumeCurve & PitchCurve on the sound you want.
minimum_rate supported unsupported This parameter is no longer supported in sound.xml. You can use VolumeCurve & PitchCurve on the sound you want.
maximum_rate supported unsupported This parameter is no longer supported in sound.xml. You can use VolumeCurve & PitchCurve on the sound you want.
minimum_speed supported unsupported This parameter is no longer supported in sound.xml. You can use VolumeCurve & PitchCurve on the sound you want.
maximum_speed supported unsupported This parameter is no longer supported in sound.xml. You can use VolumeCurve & PitchCurve on the sound you want.
initial_volume unsupported unsupported Specifies the volume at which the sound starts. (Volume is specified in 1/100dB units, with a value of 10,000 being the maximum possible volume.).
InsideConeAngle supported InsideConeAngle Inside sound cone angle in degrees.
In sound.xml InsideConeAngle="90"
OutsideConeAngle supported OutsideConeAngle Outside sound cone angle in degrees.
In sound.xml OutsideConeAngle="360"
ConePitch supported ConePitch The pitch, in degrees, of the sound cone.
In sound.xml ConePitch="50"<
ConeHeading supported ConeHeading Direction of sound cone in degrees. Zero is facing forward.

OutsideConeVolume supported OutsideConeVolume Maximum volume outside of sound cone. Volume is specified in 1/100dB units, with a value of 10000 being the maximum possible volume.
In sound.xml OutsideConeVolume="6000"

panning supported Panning Determines where the sound is placed in the stereo field:
0 : center
-10000 : full left
10000 : full right
In sound.xml FullScaleDistance="150"
