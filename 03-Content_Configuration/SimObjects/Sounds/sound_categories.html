WwisePackages

This category allows to define the names of the main package and any additional packages to be loaded with the SimObject, for sounds using the wwisedata=true attribute.

MainPackage element : Describe the name of the MainPackage that needs to be loaded by the SimObject. For an audio package to be publishable, you must write your company name as a prefix, followed by the name of your aircraft.

For exemple, the Asobo’s C152 package is set up as follows : <MainPackage Name="Asobo_C152"/>.
>IMPORTANT: All wwise events included in the MainPackage must have its name as a prefix, followed by the name described by the WwiseEvent attributes.

AdditionalPackage elements : Describe the additional packages that are used for add-ons. As for the main packages, you must write your company name as a prefix, followed by the name of the aircraft, and followed by the name of the additional content. >IMPORTANT: Every event in wwise are prefixed by the name of the MainPackage and followed by the name described in the WwiseEvent attributes whether it comes from the main package or from any additional packages. So for additional packages we recommend that you add a signature nomenclature to your WwiseEvents attributes so there won’t be any collision between the mainpackage or other additional packages. For example, by adding the name of your company, if you are not the content creator of the main package.

This category must be setup like this :

  <WwisePackages>
          <MainPackage Name="MyCompany_MyAircraft"/>
          <AdditionalPackage Name="MyCompany_MyAircraft_Additional1"/>
          <AdditionalPackage Name="MyCompany_MyAircraft_Additional2"/>
        </WwisePackages>

If this category is not specified, the name of the package loaded in game will be deduced by the path of sound.xml.

Acoustic Parameters

This section in the sound.xml, allows to set up acoustic parameters such as :

ReverberationPreset parameters :

These parameters allow to customize special sharesets for inside/outside bus. Shareset will be apply at runtime on customs auxiliary bus. The attributes for the sharesets are ShareSetInside and ShareSetOutsideparameter.

<AcousticParameters>
        <!-- the game-defined auxiliary bus in the cockit is "rev_inside_default" aux bus -->
        <Parameter Name="ReverberationPreset" ShareSetInside="shareset_inside_name" ShareSetOutide="shareset_outside_name"/>
      </AcousticParameters>

Note: AI planes do not support shareset configuration for the inside bus.

Insulation_ExitClosed and Insulation_ExitOpen :

These parameters allows to set up the sound insulation of a cockpit to attenuate outside sounds. Values are set in dB, with a range of -24dB to 0dB (no attenuation). The attributes are the following:
-Insulation_ExitClosed : the value in dB with exit door closed
-Insulation_ExitOpen : the value in dB with exit door open

<AcousticParameters>
      
        <!-- Sounds comming from outside will be attenuate by -22dB in the cockpit, if exit doors are closed -->
        <Parameter Name="Insulation_ExitClosed" Value="-22"/>
      
        <!-- Sounds comming from outside will be attenuate by -12dB in the cockpit, if exit doors are open -->
        <Parameter Name="Insulation_ExitOpen" Value="-12"/>
      
      </AcousticParameters>

Always Play Sounds

These sounds play at all times and thus do not require any parameter to set up how and when they are triggered.

<AlwaysPlaySounds>
        <!-- This sound will play all the time -->
        <Sound WwiseEvent="Always_Play_Example" WwiseData="true" />
      </AlwaysPlaySounds>

Trigger Sounds

These sounds are triggered by the simulation directly. This makes them simple to set up, but limited to logic events triggered in the code.

These sounds triggers are based on the properties of the legacy sound.cfg.

They have specific tags in the XML (GroundSounds, MiscellaneousSounds and WindSounds). Each trigger event must be set as a value in the WwiseEvent attribute.

  • GroundSounds (<GroundSounds>)

    • Continuous sound when the wheels are in contact with the ground (Loop sounds).

      • GROUND_ROLL

    • When the aircraft wheels touch the ground (One Shot sounds):

      • CENTER_TOUCHDOWN
      • AUX_TOUCHDOWN
      • LEFT_TOUCHDOWN
      • RIGHT_TOUCHDOWN

    • When a non-wheel component touches a solid element (Loop sounds):

      • FUSELAGE_SCRAPE
      • LEFT_WING_SCRAPE
      • RIGHT_WING_SCRAPE

    The correspondence between contact points and sound triggers are editable in the flight_model.cfg, in the contact_point section, on the 14th parameter for each contact point.

    By default, in a wwise data approach, each GroundSounds receives Simvar values through Wwise RTPCs. All of these RTPC are available in the WwiseSampleProject. The list of the shared RTPC can be found in the shared RTPC list.

    <GroundSounds>
        <Sound WwiseEvent="CENTER_TOUCHDOWN" WwiseData="true" />
        <Sound WwiseEvent="LEFT_TOUCHDOWN" WwiseData="true" />
        <Sound WwiseEvent="RIGHT_TOUCHDOWN" WwiseData="true" />
        <Sound WwiseEvent="GROUND_ROLL" WwiseData="true" />
      </GroundSounds>

  • WindSounds (<WindSounds>)

    As for AlwaysPlaySounds , WIND_SOUND plays at all times, but they receive physics simulation variables by default through Wwise RTPCs in the wwise data approach. All of these RTPCs are available in the WwiseSampleProject. The list of the shared RTPC can be found in the shared RTPC list.

  • MiscellaneousSounds (<MiscellaneousSounds>)
    Those sounds trigger when the corresponding event occur

    • FLAPS
    • GEAR_UP
    • GEAR_DOWN
    • EXIT_OPEN
    • EXIT_CLOSED
    • GYRO_SOUND
    • FUEL_PUMP
    • STALL_WARNING
    • OVERSPEED_WARNING_SOUND
    • AP_DISENGAGE_SOUND
    • AP_PREFLIGHT_CHECK_OVER
    <MiscellaneousSounds>
        <Sound WwiseEvent="STALL_WARNING" WwiseData="true" />
        <Sound WwiseEvent="OVERSPEED_WARNING_SOUND" WwiseData="true" />
        <Sound WwiseEvent="AP_PREFLIGHT_CHECK_OVER" WwiseData="true" /> 
      </MiscellaneousSounds>

Trigger Sounds would not require any extra settings, but they can be added if needed.

Engine Sounds

These sounds can be set up in 3 different ways depending on which sub-category they belong to:

Engine state:

These sounds are linked to the state of the engine of the SimObject. Three different engine states are differentiated: Start, On and Off, and three boolean values must thus be defined to set when the sound should be played in relation to the engine’s state.

<!--This sound will play on Engine 1 when the engine is in "Start" and "On" states --> 
      <EngineSoundStates>
          <Sound WwiseEvent="MySound" WwiseData="true" EngineIndex="1"  StateOff="false" StateStart="true" StateOn="true">
          </Sound>
      </EngineSoundStates>

Engine transition:

The sound triggers when the engine goes from one state to another. A StateTo and a StateFrom values will be needed for the sound to be playable, each can be set to Off, Start or On.

<!--
      This sound will play on Engine 2, when the engine goes from Start
      to On
      -->
      <EngineSoundTransitions>
        <Sound WwiseEvent="MySound" Wwisedata="true" StateFrom="Start" StateTo="On"  EngineIndex="2"/>
      </EngineSoundTransitions>

Engine presets:

These are set to specific play conditions based on their WwiseEvent, and mimic the behavior of the respective presets in the legacy sound.cfg format. The wwise event will be named accordingly to the EngineIndex (X) and the FSX legacy naming.

XML WwiseEvent Behavior Event Name in Wwise Loop or OneShot
JetWhine Played all the time engX_jet_whine Loop
NonCombustion When engine is “off” engX_non_combustion Loop
Starter When the engine is “starting” engX_starter Loop
CombustionStart When the engine goes from “starting” to “on” engX_combustion_start OneShot
Combustion When engine is “on” engX_combustion Loop
Propeller When engine is “on” engX_prop Loop
Rotor When engine is “on” engX_rotor Loop
Shutdown When the engine goes from “on” to “off” engX_shutdown OneShot 
<EngineSoundPresets>
        <!-- This sound will play the wwise event "Play_SimObjectName_eng2_combustion" on the 2d engine-->
        <Sound WwiseEvent="Combustion"   Wwisedata="true" EngineIndex="2"/>
      </EngineSoundPresets>

By default, in a wwise data approach, each EngineSoundPresets event receives Simvar values through Wwise RTPCs. All of these RTPC are available in the WwiseSampleProject. The list of the shared RTPC can be found in the shared RTPC list.

Specific EngineSoundPresets attributes : - RPMMin : this setting allows to set the minimum RPM threshold ( GENERAL ENG RPM simvar) from which the engine loop stops, and from which the Shutdown sound plays. Its purpose is to get a better transition between the loop and the engine shutdown when the engine state goes off.

  <EngineSoundPresets>
      
          <!-- This sound will stop if the engine state goes to 'off' and if the 3rd engine is below the 675rpm threshold. -->
          <Sound WwiseEvent="Combustion" ConeHeading="225" RPMMin="675" WwiseData="true" EngineIndex="3"/>
      
          <!-- This sound will play if the engine state goes to off and if the 3rd engine rpm is below the 675rpm threshold. --> 
          <Sound WwiseEvent="Shutdown" ConeHeading="225" RPMMin="675" WwiseData="true" EngineIndex="3"/>
      
        </EngineSoundPresets>
  • CustomSuffix : In Wwise, each event has its own name and ID. This setting allows to use as many EngineSoundPresets as needed.
  <EngineSoundPresets>
      
          <!-- This sound will play the wwise event "Play_SimObjectName_eng3_combustion_from_left" on the 3rd engine.-->
          <Sound WwiseEvent="Combustion" CustomSuffix="from_left" ConeHeading="225" RPMMin="675" WwiseData="true" EngineIndex="3"/>
      
          <!-- This sound will play the wwise event "Play_SimObjectName_eng3_shutdown_from_left" on the 3rd engine.--> -->
          <Sound WwiseEvent="Shutdown" CustomSuffix="from_left" ConeHeading="225" RPMMin="675" WwiseData="true" EngineIndex="3"/>
      
        </EngineSoundPresets>

Event Id Sounds

Much like trigger sounds, these sounds are set to receive events from the simulation. But in the case of EventId sounds, each event is paired with an associated value. They require an EventId setting to specify the event that they have to react to.

They can be set to act every time the event is fired, or only when the event changes its value using the setting ChangedOnly=“true”/“false”. Its default value is false.

<EventIdSounds>
      
        <Sound WwiseEvent="Sound_For_Test_1" Wwisedata="true" EventId="KEY_TEST_EVENT_1"/>
        <!-- This sound will play every time the event is fired -->
      
        <Sound WwiseEvent="Sound_For_Test_2" Wwisedata="true"  EventId="KEY_TEST_EVENT_2" ChangedOnly="true"/>  
        <!--This sound will play only when the event fired has a different value than the last time it was fired-->  
      
      </EventIdSounds>

For a list of all event ids, see the EventId List.

Sim Var Sounds

These sounds react to the changes in values in the simulation variables directly. They can either be played when the watched value changes, or when the value reaches a specific range. A Range is defined by an LowerBound value and a UpperBound value. These values default to -∞ and +∞.

These sounds will need a SimVar setting, followed by Units and Index. These three settings will define the simulation variable value that will be watched.

<SimVarSounds>
      
        <!--This sound will play every time the simvar value changes -->
        <Sound WwiseEvent="Sound_For_Test_1" Wwisedata="true" SimVar="SIMVAR TEST 1" Units="PERCENT" Index="0"/>
      
        <!--This sound will play whenever the simvar value is between 10 and 15, and when it's over 25. -->
        <Sound WwiseEvent="Sound_For_Test_2" Wwisedata="true" SimVar="SIMVAR TEST 2" Units="PERCENT" Index="0">
          <Range LowerBound="10.0" UpperBound="15.0"/>
          <Range LowerBound="25.0" />
        </Sound>
        
      </SimVarSounds>

In addition to the Range parameters, the Require element allows to add other Simvar or LocalVar (see below) conditions on a sound to be played.

<SimVarSounds>
        <!--This sound will play if :
        - SIMVAR TEST 1 value is between 10 and 15, and when it's over 25,
        - And if SIMVAR TEST 2 value is above 60, 
        - And if SIMVAR TEST 3 value changes. -->
      
        <Sound WwiseEvent="Sound_For_Test_2" Wwisedata="true" SimVar="SIMVAR TEST 1" Units="PERCENT" Index="0">
          <Range LowerBound="10.0" UpperBound="15.0"/>
          <Range LowerBound="25.0" />
          <Requires SimVar="SIMVAR TEST 2" Units="PERCENT" Index="0">
                <Range LowerBound="60" />
          </Requires>
          <Requires SimVar="SIMVAR TEST 3" Units="PERCENT" Index="0">
          </Requires>
        </Sound>
        
      </SimVarSounds>

Instead of using SimVar, you can also set up local variables as conditions to play a sound. The LocalVar attribute works the same way as the Simvar attribute, except it has no Units and Index attributes.

<SimVarSounds>
        <!--This sound will play whenever the Local variable value is between 0.1 and 0.15, and when it's over 0.25. -->
      
        <Sound WwiseEvent="Sound_For_Test" Wwisedata="true" LocalVar="MyLocalVar TEST 1" >
          <Range LowerBound="0.1" UpperBound="0.15"/>
          <Range LowerBound="0.25" />
          <Requires SimVar="MyLocalVar TEST 2" Units="PERCENT" Index="0">
            <Range LowerBound="1" />
          </Requires>
        </Sound>
        
      </SimVarSounds>

In addition to previous settings, ContinuousStopDelay can be set to a value in seconds, and will delay the stop of the continuous sound linked to the simvar after the condition for stopping has been reached.

For a list of all simulation variables with their units and indices, visit the SimVar List section.

Animation Sounds

These sounds are triggered on Animations implemented in the Simobjects model xml files, and use ModelBehaviors element. Their main purpose is to play buttons and switches sounds in the cockpit, but they can also be used elsewhere, e.g. on flight control sounds, levers, Jetways animations…etc.

To play a sound, the Animation must have the following attributes (in the animation xml file) : - AnimationTriggers : This is the parent element. It should contains the Animation attribute whose value is the name of the animation, so that the sound plays on it.

  • EventTrigger : This element is used to describe how the sound is played on the animation timeline. Here are the useful attributes for audio :
    • NormalizedTime : Normalized time of the animation from which the sound will play. It’s going from 0 to 1. This attribute must be used alone, without a Count or Frame attribute.
    • Count : This attribute divides the animation timeline in equal segments by the value written . Sounds will play at the midlle of each segment. e.g. 
      • If the value is 2 the sound will play when the animation timeline reaches 25% and 75%.
      • If the value is 5 the sound will play when the animation timeline reaches 10%, 30%, 50%, 70 and 90%.
        This attribute must be used alone, without a NormalizedTime or Frame attribute.
    • Frame : the frame of the animation where the sound will be played. this attribute must be used alone, without a Count or NormalizedTime attribute.
    • Direction : Determines the direction of the animation timeline in which the sound will be played. So the sound can play when the animation goes in the :
      • Forward direction,
      • Backwarddirection,
      • Or in Both direction.
  • SoundEvent child element that contains these attributes :
    • WwiseEvent : it’s the same attribute that must also be set in the sound.xml.
    • Action : Play or Stop value. The stop is useful for loops.

Here is a basic declaration of an animation sound in both XML files (animation + sound):

Animation XML file sample

 <!--MySound will play each time the animation timeline reaches 25% and 75% -->
          <AnimationTriggers Animation="MyAnimationName">
              <EventTrigger Count="2" Direction="Both">
                  <SoundEvent WwiseEvent="MySound" Action="Play"/>
              </EventTrigger>
          </AnimationTriggers>
 <!--MySound will play each time the animation timeline reaches 10% when going forward, and each time the animation timeline reaches 90% when going backward-->
          <AnimationTriggers Animation="MyAnimationName">
              <EventTrigger NormalizedTime="0.1" Direction="Forward">
                  <SoundEvent WwiseEvent="MySound" Action="Play"/>
              </EventTrigger>
              <EventTrigger NormalizedTime="0.9" Direction="Backward">
                  <SoundEvent WwiseEvent="MySound" Action="Play"/>
              </EventTrigger>
          </AnimationTriggers>

Sound XML file sample

<AnimationSounds>
       
         <!--This sound will play every time the event "MySound" is sent through the animation-->
         <Sound WwiseData="true" WwiseEvent="MySound" ViewPoint="Inside"/>
      
      </AnimationSounds>

For more information about Model Behaviors , see the Model Behavior section.

Avionic Sounds

Sounds in this category can be triggered through html gauge or panel.xml logics. Like AnimationSounds, they must be referenced in sound.xml, and share the same WwiseEvent in both XML files.

PANEL XML file sample

<Alert>
          <Type>SoundOnly</Type>
          <SoundEvent>voice_alert_500ft</SoundEvent>
          <Condition>
              <StateMachine>
                  <State id="Under" value="0">
                      <Transition to="Over">
                          <GreaterEqual>
                              <Simvar name="PLANE ALT ABOVE GROUND" unit="feet"/>
                              <Constant>500</Constant>
                          </GreaterEqual>
                      </Transition>
                  </State>
                  <State id="Over" value="0">
                      <Transition to="Alerting">
                          <Lower>
                              <Simvar name="PLANE ALT ABOVE GROUND" unit="feet"/>
                              <Constant>500</Constant>
                          </Lower>
                      </Transition>
                  </State>
                  <State id="Alerting" value="1">
                      <Transition to="Under">
                          <Constant>1</Constant>
                      </Transition>
                  </State>
              </StateMachine>
          </Condition>
      </Alert>

Sound XML file sample

  <AvionicSounds> 
            <Sound WwiseData="true" WwiseEvent="voice_alert_500ft" />
        </AvionicSounds>
