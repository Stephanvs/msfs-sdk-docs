Common Sound attributes

This settings are shared between all the sounds, they help define the most important elements of a sound. They can be used as an attribute of a Sound element in both Wave Data and Wwise Data approach.

  • WwiseEvent

    This setting is a must have attribute in both wwisedata and wavedata approaches. It defines the name of the wwise event to play in a sound element. As seen in the sound_categories page, EnginesSoundPreset are the exception.
    In the WaveData approach, this attribute defines the wwise event as a slot for the .wav sound to be routed. These slots are limited to the existing ones. Please have a look to the xml_wavedata page to have an overview of all the available wwise event slots in a WaveData approach.

  • WwiseData

    This setting defines if the sound is a .wav file in the sound folder or audio included into the Wwise Sounbank. It can be set to :

    • WwiseData=“true” wwise data approach: the sound is included into the SimObject .PC.PCK file package,
    • WwiseData=“false”wave data approach: the sound is a .wav file located in the SimObject sound folder.

  • ViewPoint

    This setting specifies if the sound can be heard from outside the SimObject, inside the SimObject or both. It can be Inside, Outside or All, the default value for this is All.

    • ViewPoint=“Inside” sound will play inside the cockpit,
    • ViewPoint=“Outside” sound will play outside the cockpit,
    • ViewPoint=“All” default value, the sound will play on both viewpoint.

  • FadeOutType

    This setting allows for sounds to fade out when asked to stop playing. It can be set to :

    • FadeOutType=“0” for None,
    • FadeOutType=“1” for Linear,
    • FadeOutType=“2” for Sine,
    • FadeOutType=“3” for Reciprocal Sine.

    The default value is 0.

  • FadeOutTime

    Only if FadeOutType is not None this can be set. It represents the time in seconds that the sound will take to fade out.

  • BreakOnStop

    If true, when a stop is called on a sound, it will plays until its end. It can be set to :

    • BreakOnStop=“true” The sound will play until it reach its end, and then stop,
    • BreakOnStop=“false” the default value, the sound stops when the stop is called.

  • Continuous

    This boolean value allows this user to override the default value which defines whether a sound is played in a loop, or as a one-shot sound. The default value depends on the sound category and the sound type. It can be set to :

    • Continuous=“true” if the sound is a loop
    • Continuous=“false”if the sound is a one-shot

  • ContinuousStopDelay

    ContinuousStopDelay can be set to a value in seconds, and will delay the stop of a continuous sound. linked to the simvar after the simvar itself has stopped changing over time.

    • ContinuousStopDelay=“0.5” means the sound will be delayed by 0.5 seconds.

  • ApplyCockpitReverb

    This boolean allows to active or deactive the sending to the game-defined bus defined in the ReverberationPreset attribute, when inside cockpit. It can be set to :

    • ApplyCockpitReverb=“true” default value, the sound will be sent to the game-defined bus
    • ApplyCockpitReverb=“false”if you don’t want the sound to be sent into the game-defined bus.

  • NodeName

    This setting allows to specify a node from the model to attach the sound to. If not set or if the node is not found, the system will attach the sound to the main node of the SimObject. This attribute is not available on engines sounds (EngineSoundPresets EngineSoundStates and EngineSoundTransitions) as engines sounds are attached to the position of the engines specified in the engine configuration file ([generalenginedata]engine.cfg) thanks to the EngineIndex attribute.

    • NodeName=“TheNodeName”

  • ConePitch and ConeHeading

    Settings to change the direction the sound is facing (in degrees 0-360). By default both are 0.

    • ConePitch = The pitch, in degrees, of the sound cone.
    • ConeHeading = Direction of sound cone in degrees. Zero is facing forward.

  • CancelConeHeadingWhenInside

    Boolean value allowing the cancellation of the ConeHeading setting whenever the simulation’s ViewPoint is Inside. It can be set to :

    • CancelConeHeadingWhenInside=“true” is the default value
    • CancelConeHeadingWhenInside=“false”if you want to cancel Coneheading value.

  • CustomSuffix

    This attribute only works with WwiseData=“true” attribute. It allows to add additional suffixe to any Wwise events. This way Trigger sounds can be doubled for the addition of new layers, or be replaced within an additional package.

WwiseRtpc Attributes

The WwiseRtpc child element and its attributes can only be used in a Wwise Data approach Wwisedata=“true”.

This sends simulation variable directly to Wwise. It requires a SimVar or LocalVar values and the name of the RTPC in Wwise.

A sound can have as many RTPC values as needed.

Here are the available parameter that can be set on WwiseRtpc setting

  • SimVar : The simulation variable from which the RTPC will be send to wwise. It requires Units and Index attributes.
  • LocalVar : The local variable from which the RTPC will be send to wwise. Local variable do not require any Units and Index attributes.
  • RTPCName : The name of the game parameter in Wwise.
  • Derived : Boolean that allow to derived a Simvar value,
  • RTPCAttackTime: Setting that allows to customize the smooth time during ascending movement. The smaller the value, the faster the target will be reached.
  • RTPCReleaseTime : Setting that allows to customize the smooth time during descending movement. The smaller the value, the faster the target will be reached.
<Sound WwiseEvent="wwisertpc_test" WwiseData="true">
      
          <!-- This simvar will be sent directly to Wwise -->
          <WwiseRTPC SimVar="SIMVAR TEST 1" Units="PERCENT" Index="0" RTPCName="SIMVAR_TEST_1"/>
      
          <!-- This localvar will be sent directly to Wwise -->
          <WwiseRTPC LocalVar="LOCALVAR TEST 1" Units="PERCENT" Index="0" RTPCName="LOCALVAR_TEST_1"/>
      
          <!-- This simvar will be derived before being sent to Wwise -->
          <WwiseRTPC SimVar="SIMVAR TEST 2" Units="PERCENT" Index="0" Derived="true" RTPCName="SIMVAR_TEST_2_DERIVED"/>
      
          <!-- This simvar will be sent to Wwise with a smooth during ascending periods  -->
          <WwiseRTPC SimVar="SIMVAR TEST 1" Units="PERCENT" Index="0"  RTPCName="SIMVAR_TEST_1" RTPCAttackTime="0.5"/>
      
          <!-- This simvar will be derived and smoothed during descending periods before being sent to Wwise   -->
          <WwiseRTPC SimVar="SIMVAR TEST 1" Units="PERCENT" Index="0" Derived="true" RTPCName="SIMVAR_TEST_1" RTPCReleaseTime="0.5" />
        
      </Sound>

Wwisedata=“false” specific Sound attributes

The Wwisedata=“false” specific attributes come from the legacy sound.cfg properties, porting from the old format to the xml format.These attributes are available only when using the Wave Data Approach : WwiseData=“false”

  • FileName

    When WwiseData is set to false, FileName is required. It defines the name of the .wav file to play. The .wav extension should not be specified. If FileName attribute have comma separated filenames (e.g. FileName=“encrash1,encrash2”),the simulator code will randomly choose to play one of the listed .wav files. In sound.xml:

    • FileName=“Mysound01,Mysound02,Mysound03”

  • FullScaleDistance

    The distance, in meters, away from the sound source, when the sound starts to attenuate. If not 0, distance attenuation will be active, using the specified FullScaleDistance factor. The greater the value the further the sound has to be to attenuate. By default it is 0 meaning no attenuation will be calculated.

    • FullScaleDistance=“150” the sound will starts to attenuate over 150m distance

  • InsideConeAngle and OutsideConeAngle

    This settings defines three regions of the cone in 3D: the regions inside the InsideConeAngle will not be attenuated, the regions between the angles will attenuate gradually until it reaches OutsideConeAngle where it will default to the OutsideConeVolume. These values are in degrees (0-360). If not set, both angles are 360, disabling all possible attenuations depending on the angles.

    • InsideConeAngle=“0” The sound will be at full volume in front of the object
    • OutsideConeAngle=“180” Behind the object the sound will take OutsideConeVolume value.

  • OutsideConeVolume

    Defines how much a sound is attenuated outside the outside cone. It can go from 10000(full volume) to 0(no sound), by default it is 0. The curve attenuation is logarithmic.

    • OutsideConeVolume=“6000”

  • Panning

    Defaults as 0 and is ranged between -10000 and 10000. It moves the sound from the left to the right “point of view” of the system as if it was 3D.

    • Panning=“10000” means sound will play in 2d on the right channel
    • Panning=“-10000” means sound will play in 2d on the left channel

  • MuteOnVPChange

    This boolean allows, for continuous sounds, to be muted instead of stopped when the specified viewpoint in which they should play is not longer presented by the simulation. By default, it is false for all sounds except for the Gyro and Fuel Pump sounds.

    • MuteOnVPChange=“false” is the default value
    • MuteOnVPChange=“true” the sound is muted instead of stopped when the specified viewpoint is no longer true.

### VolumeCurve and PitchCurve

PitchCurve and VolumeCurve settings can be defined as sound child elements.

These curves were mainly inherited from the vparams and rparams. They define behavior for the pitch and/or the volume for a specific Wave sound. Unlike vparams and rparams settings in sound.cfg, there is no curve point limitation with PitchCurve and VolumeCurve.

They need a simulation variable (id, units and index) and a set of points to define the volume/pitch output for the values of the simulation variable.

The Output value for the PitchCurve is in frequency ratio. A value of 1.0 specifies that the sound file is played at unity pitch. A value of 2.0 specifies that the file is played an octave higher and twice as fast.A value of 0.5 specifies that the file is played an octave lower and twice as slower.

The Output value for the VolumeCurve is in Percent, clamped between 0.0 and 100.0. The units are linear, with a value of 50 meaning -3dB attenuation, and 0 meaning silence.

Multiple pitch and/or volume curves are allowed and will be combined in code before being used with the sound.

<Sound WwiseEvent="volume_pitch_test" Wwisedata="false" FileName="volume_pitch_test">
        <PitchCurve SimVar="SIMVAR_TEST_PITCH" Units="PERCENT" Index="1">
          <Point SimValue="0.000" Output="0.5" />
          <Point SimValue="100.000" Output="1.551" />
        </PitchCurve>
        <VolumeCurve SimVar="SIMVAR_TEST_VOLUME" Units="PERCENT"
        Index="1">
          <Point SimValue="0.005" Output="0.0" />
          <Point SimValue="0.127" Output="3.2" />
          <Point SimValue="0.197" Output="16.8" />
          <Point SimValue="0.238" Output="76.8" />
          <Point SimValue="0.336" Output="78.4" />
          <Point SimValue="0.405" Output="18.4" />
          <Point SimValue="0.590" Output="0.0" />
          <Point SimValue="0.630" Output="0.0" />
        </VolumeCurve>
      </Sound>
