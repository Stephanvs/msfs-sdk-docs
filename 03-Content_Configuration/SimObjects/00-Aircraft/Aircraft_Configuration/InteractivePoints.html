Interactive Points section

  • Basics
  • Various fields of interactive_point.*
    • About type, Position, and Orientation
    • About Jetway values
  • ModelBehaviors and SimVars related to interactive points

The [INTERACTIVE POINTS] section is currently located in the file flight_model.cfg (see here for more details on this file). This might change in future, as they have no actual impact on the flight model.

Basics

Interactive Points in MSFS represent an extension of the [EXITS] from FSX. They are used to define position of various doors of the aircraft (regular, emergency, and cargo doors), as well as some other points to interact with Airport Services, such as the end of a fuel hose to interact with a FuelTruck, or the end of an eletrical cable to interact with a GroundPowerUnit vehicle.

The [INTERACTIVE POINTS] should contain a line number_of_interactive_points = X with the number of interactive points instead of X, followed by X lines interactive_point.Y = with increasing numbers going from 0 to X-1 instead of Y.

Interactive Points of an aircraft can be edited via the Aircraft Editor.

Various fields of interactive_point.*

Each line of interactive_point has 14 comma-separated fields, interactive_point.* = A, B, C, D, E, F, G, H, I, J, K, L, M, N, where each letter here has its correspondance in the following table:

Idx Name in editor Description SimVar
A Open Close Rate Percent per second of animation of the interactive point (mostly for doors) -
B Pos (3rd value) Coordinate in feet of the point relative to aircraft CG, on the back-to-front axis INTERACTIVE POINT POSZ
C Pos (1st value) Coordinate in feet of the point relative to aircraft CG, on the left-to-right axis INTERACTIVE POINT POSX
D Pos (2nd value) Coordinate in feet of the point relative to aircraft CG, on the bottom-to-top axis INTERACTIVE POINT POSY
E Type Integer corresponding to an enum, determining the type of the point (see below for more details) INTERACTIVE POINT TYPE
F Orientation (1st value) Pitch, in degrees, of the point orientation (0 means horizontal) INTERACTIVE POINT PITCH
G Orientation (2nd value) Bank, in degrees, of the point orientation (currently unused, please set 0 here) INTERACTIVE POINT BANK
H Orientation (3rd value) Heading, in degrees, of the point orientation (0 means same heading as the aircraft) INTERACTIVE POINT HEADING
I Jetway Left Bend In percentage (0 to 100), see the dedicated subsection INTERACTIVE POINT JETWAY LEFT BEND
J Jetway Left Deployment In degrees, see the dedicated subsection INTERACTIVE POINT JETWAY LEFT DEPLOYMENT
K Jetway Right Bend In percentage (0 to 100), see the dedicated subsection INTERACTIVE POINT JETWAY RIGHT BEND
L Jetway Right Deployment In degrees, see the dedicated subsection INTERACTIVE POINT JETWAY RIGHT DEPLOYMENT
M Jetway Top Horizontal In percentage (-100 to 100), see the dedicated subsection INTERACTIVE POINT JETWAY TOP HORIZONTAL
N Jetway Top Vertical In percentage (-100 to 100), see the dedicated subsection INTERACTIVE POINT JETWAY TOP VERTICAL

More information on the associated simvars are available in the simvars list.

About type, Position, and Orientation

The following types are available:

Type Description
0 Main exit (i.e. regular aircraft door)
1 Cargo exit/door
2 Emergency exit
3 Fuel hose
4 Ground Power cable
99 Unknown (used for errors)

For the first three types, the interactive point should be positioned at the bottom of the door (in the middle of it for other directions). The orientation should point horizontally towards the exterior of the door (i.e. perpendicular to the door itself). This position and orientation will be used by Jetways, BoardingRamps, CateringTruck, and (for cargo) BaggageLoaders, in order to align properly.

For an aircraft to properly interact with a FuelTruck, it should have a fuel hose Node as part of its model, hidden by default thanks to ModelBehaviors (see section below), preferably on the left side of the aircraft. The associated interactive point of type 3 should then be placed at the end of the hose (on the fueltruck side), in the center of te circle (assuming a cylindrical hose). The orientation should point towards the exterior of the hose, in the same direction as the last part of the hose.

For an aircraft to properly interact with a GroundPowerUnit, it should have an electrical cable Node as part of its model, hidden by default thanks to ModelBehaviors (see section below), preferably on the right side of the aircraft, close to its nose. The associated interactive point of type 4 should then be placed at the end of the cable (on the groundpowerunit side), in the center of te circle (assuming a cylindrical cable). The orientation should point towards the exterior of the cable, in the same direction as the last part of the cable.

Note that, when using the Aircraft Editor, interactive points are represented on the aircraft, with an arrow starting from their position that indicates their orientation.

About Jetway values

The last six values of an interactive point are used to properly animate the hood of a jetway, in order to best match the shape of the aircraft. This also means those values are only useful for interactive points of type 0, and only for aircrafts suitable for jetway interaction (defined in the [SERVICES] section of aircraft.cfg).

Those six values are matched with the six animations for jetway hood. The left and right deployment values are angles in degrees to incline the base of the jetway hood (with 0 being the default vertical position). The left and right bend values are percentage of the bending animation (0 to 100, where 0 is the default state). The top horizontal and vertical values are signed percentage of the corresponding animations (-100 to 100, where 0 is the default state).

To properly define them for your aircraft, there is sadly no magic recipe, and you will have to tweak those values until the result is satisfying. The recommended way to do so is to start a flight on a gate with your aircraft, call the jetway with the shortcut input TOGGLE JETWAY (default to ctrl+J, can be changed in the controls option menu), examine the current result of hood animation, retract the jetway with the shortcut input, edit interactive point values, save them and reloads your aircraft sim, then calls the jetway again and repeat (issues can happen if jetway is still connected to aircraft with the latter is reloaded).

The good thing about those values is that any jetway created by any compagny will give similar results on your aircraft, as long as they use the same animation conventions.

To give an example, the default Airbus A320 Neo in MSFS has the following last six values for its interactive point of the front-left door: 72, 16, 85, 3, -2, 33. Corresponding, in order, to: left bend, left deployment, right bend, right deployment, top horizontal, top vertical. And the Boeing 747-8i has the following values : 25, 4, 35, 1.5, 0, 0.

ModelBehaviors and SimVars related to interactive points

Each interactive point simvar has to use an index corresponding to the index of the interactive point in the cfg (starting at 0). In addition to the simvars listed here and representing the constant values of interactive points, another simvar, INTERACTIVE POINT OPEN, is available. This simvar is a percentage which is 0% when the corresponding interactive point door is closed, or hose/cable is not visible, and 100% when the door is open, or hose/cable is fully deployed.

All those simvars can be used directly, or you can also use the following ModelBehavior templates, by using this include: <Include Path="Asobo\Exterior.xml"/>.

The template ASOBO_DOOR_InteractivePoint_Template is used to play an animation of a door opening, matching the simvar for the associated interactive point. It can also be used for hose/cable deployment animations. It takes two parameters, <ID> which is the index of the corresponding interactive point, and <ANIM_NAME> containing the name of the corresponding animation. Note that the animation’s first frame should be with the door closed (or hose/cable not visible).

The template ASOBO_FuelHose_InteractivePoint_Template is used when no animation is available, to hide/show a hose/cable Node in the model, based on the associated simvar value. It takes two parameters, <ID> which is the index of the corresponding interactive point, and <NODE_ID> which is the name of the Node in the model. Note that the Node will be hidden when INTERACTIVE POINT OPEN is 0 (by default) and shown for values strictly greater than 0.
