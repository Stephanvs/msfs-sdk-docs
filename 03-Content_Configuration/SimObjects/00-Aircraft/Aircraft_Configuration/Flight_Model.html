Introduction

The flight model configuration file is an optional file; the sections described here may be defined in the aircraft.cfg as well. If used, it should be named flight_model.cfg and located in the root directory of the aircraft, next to the aircraft.cfg.

This configuration file defines the flight dynamics model: parameters that tune the flight simulation to match the aircraft’s realistic physical behavior.

This documentation is not yet complete. For now, please look at the ESP documentation‘s Aircraft Configuration File for missing sections’ parameters. Most (if not all) of the parameters described there are valid for Microsoft Flight Simulator.

Sections

[VERSION]

This version section provides information on version of the configuration file. In Microsoft Flight Simulator the major version should always be at least equal to 1.

Property Description Type Unit Range Mandatory
major Major CFG file version number Integer Number >= 0 Yes
minor Minor CFG file version number Integer Number >= 0 Yes

[WEIGHT_AND_BALANCE]

Datum Reference and positioning

Most position parameters are given relative to the datum reference point for the aircraft. The convention for positions is positive equals forward, to the right, and vertically upward. Units are in feet, unless mentioned otherwise.

3-D coordinates are given with respect to this referential and in this order:

  1. z (longitudinal) coordinate
  2. x (lateral) coordinate
  3. y (vertical) coordinate

The datum reference point itself is specified in this section.

The only parameters that are not relative to the datum reference are the parameters for the manual longitudinal positioning of the center of lift (CoL), compute_aero_center=0 and aero_center_lift. See the flight-model documentation.

[CONTACT_POINTS]

We need to define points in the aircraft body referential frame which are likely to come in contact with the ground. This is used for aircraft positioning on ground and also for crash simulations. A contact point is defined with a 1D-table with 16 floats as follows:

point.0 = 1, -10, 0.0, -14.5, 1600, 0, 2.0, 70.0, 0.5, 3.5, 0.900, 9.0, 8.0, 0, 220, 250;

For the example above: Point.0 = wheel, -10 on x-axis, 0 on y-axis, 0 on z-axis, 1600 Impact damage, no brake, 2 ft wheel radius, 70 degree steering, 50% compression, 3.5ft max compression, 90% compression, etc.
