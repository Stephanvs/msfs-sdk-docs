An <Airport> is a modular structure meant to describe both the visual and logical aspects of the airport. In its most simple form, it is only containing a <Runway> but can, in its most complex forms, contain a large assembly of the structures described in the following sections, in order to represent the complexity of an airport. Due to the modular approach of the airport loading system, it is possible to separate different data types into multiple files within a package or even different packages. For example, it is possible to have the taxiway logic which is necessary for AI planes to move on the airport in a different file than the taxiway signs which are for visual purposed only.

An <Airport> may contain also navigation related data such as Approach, Departure or Arrival procedures but those will be discussed in their own specific documentation related to Navigation Data as they are overall independant from the rest of the airport data.

An <Airport> has the following attributes, mainly to describe where it is and information displayed on the world map.

Attribute Description Type Required
lat Latitude of the airport reference point Latitude Yes
lon Longitude of the airport reference point Longitude Yes
alt Altitude of the airport reference point Altitude Yes
altType Either “GEOID” or “ELLIPSOID”, it is used to determine the referential for the alt field Enum No
ident Unique identifier that is used to represent the airport, must be 3 or 4 characters long String Yes
name Name of the airport, displayed on the world map String No
city Name of the closest city of the airport, displayed on the world map String No
magvar Magnetic variation, the angle difference between the magnetic north and true north. Negative for value to the east and positive for value to the west AngleDeg No
isOnTIN Used to get a different ground materials behavior in certain area Boolean No
onlyAddIfReplace Only adds the airport if it was already present in another package. Relevant if the data contained within <Airport> is only complementary and not meant to be used on its own Boolean No
applyFlatten Generate flattening rectangles on the airport based on runways altitude and inclination Boolean No
starAirport Puts the airport forward on the world map Boolean No
closed Stop the airport from being accessible or interactive in any interface such as the world map, the instruments or the ATC system. The airport is only going to be rendered in the simulator. Boolean No

When doing work on an already existing airport in the simulator, it is necessary to be aware that by default the behavior is to add the new set of data to the existing airport not to replace the corresponding structures. If the objective is to replace the previous set of data, then it is necessary to use a <DeleteAirport> structure to describe what is to be replaced.

DeleteAirport

Since most of the airports that are going to be edited are preexisting airport in the simulator, it is important to understand the behavior behind the <DeleteAirport> structure.

When loading packages that contains airport, the simulator will load the version of an airport from each package, incrementaly adding the data of each package to the data contained in the previous ones. In case that is not what the user wants, here is a description of the attributes that can remove previously existing data.

Runway

In the same way as the <Airport> structure, a <Runway> structure is also modular structure in order to represent the variety of runways that might be encountered in the world. In its minimal configuration, it is necessary to have two <RunwayStart> structures has child to be usable. Even though a runway is one physical object, it is consituted with two logical ends that share the physical space, referenced as PRIMARY or SECONDARY in the following structures. The PRIMARY one is the one that is reference in the attributes number and designator of the <Runway>.

Jetway

Each jetway must be associated with a parking (which should be a Gate), and each parking can have at most one jetway associated to it.

A <Jetway> can contain one or two SceneryObject, where one must be an interactive jetway object, and the optional second one is an link object connecting the interactive jetway with a building. Those links are especially useful when using auto-generated buildings and not handcrafted SceneryObject buildings.

Details on jetway models and link models are available in a dedicated chapter.

Jetway XML definition

A <Jetway> element has the following attributes (identifying the associated parking) and elements (defining the jetway objects):

Attribute Description Type Required
gateName Name of the associated parking Enum Yes
parkingNumber Number of the associated parking Positive Integer Yes
suffix Suffix of the associated parking Enum Yes
Element Description Quantity
<SceneryObject> Jetway object, either the interactive jetway or the link. 1-2

Each <SceneryObject> must have one element, either <SimObject> or <LibraryObject>, and also has the following attributes:

Attribute Description Type Required
lat Latitude of the object Latitude Yes
lon Longitude of the object Longitude Yes
alt Altitude of the object Altitude Yes
pitch Pitch of the object AngleDeg Yes
bank Bank of the object AngleDeg Yes
heading Heading of the object AngleDeg Yes
altitudeIsAgl True if the altitude is relative to the ground (AGL = Above Ground Level), False if the altitude is relative to the mean sea level. Default to True. Boolean Np

Then, a <SimObject> element describes the model of interactive jetway used, while a <LibraryObject> element describes the model of link used. Note that if a Jetway has one SceneryObject, then it should contain a SimObject, and if a Jetway has two SceneryObject elements, then one should contain a SimObject and the other a LibraryObject.

A <SimObject> has the following attributes:

Attribute Description Type Required
containerTitle Title of the SimObject to be used as interactive jetway String Yes
scale Scale of the model, to make it larger (> 1.0) or smaller (< 1.0). Float Yes

A <LibraryObject> has the following attributes:

Attribute Description Type Required
name GUID of the SceneryObject to be used as jetway link GUID Yes
scale Scale of the model, to make it larger (> 1.0) or smaller (< 1.0). Float Yes
