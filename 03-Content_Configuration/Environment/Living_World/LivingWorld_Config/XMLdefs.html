Here are defined the specifications of the Living world configuration file.

<SimBase.Document>

Base of an xml document.

Element Description Quantity
<LivingWorld.LivingWorldSettings> Mandatory base element 1

<LivingWorld.LivingWorldSettings>

Base of a Living World Configuration document. Must be contained in <SimBase.Document>.

Element Description Quantity
<GeneralSettings> A set of general settings for living world 0-1
<LivingWorldRegion> Living world description of one of the 26 different regions of the world 0-26

<GeneralSettings>

Contains some General settings of the living world, without limits to regions.

Element Description Quantity
<AmbientVehiclesQuantityByParking> Determines the number of Ambient vehicles generated in an airport, based on the number of parkings of this airport 0-1
<ParkedAircraftsQuantityByParking> Determines the number of Parked Aircrafts generated in an airport, based on the number of parkings of this airport 0-1
<FrequencyOfServicesOnEmptyParkings> How often Airport Services are spawned around empty parkings 0-1
<RatioOfAmbientVehiclesThatAreImmobile> How many of the Ambient vehicles never leave their initial parking 0-1

<AmbientVehiclesQuantityByParking>

Determines how many Ambient vehicles will be generated on each airport. The number and type of parkings on the airport correlates with the type and size of the airport, and thus the number of vehicles to generate.

Element Description Quantity
<ParkingEntry> Entry with type of parking and associated number of Ambient vehicle Any

<ParkedAircraftsQuantityByParking>

Lists parking types, and determines how likely it is to generate a Parked Aircraft on each such parking, and how much best matching of parking radius impacts the model of aircraft chosen.

Element Description Quantity
<ParkingEntryWeighted> Entry with type of parking and associated number of Parked Aircraft Any

<FrequencyOfServicesOnEmptyParkings>

Determines how many of the parkings that are not occupied by aircrafts will still generate services vehicles around them to “populate” the airport.

Attribute Description Type Required
Rate Rate (between 0.0 and 1.0) of empty parkings that will have Service vehicles spawned around them Float Yes

<RatioOfAmbientVehiclesThatAreImmobile>

Determines how many of the Ambient vehicles will stay immobile instead of moving around the airport.

Attribute Description Type Required
Rate Proportion (between 0.0 and 1.0) of Ambient Vehicles that will never move and just stay at their designated parking Float Yes

<ParkingEntry>

A parking type with associated number of Ambient Vehicles to spawn per parking of this type on an airport.

Attribute Description Type Required
ParkingType Enum (see the TaxiwayParkings) for a type of parking Enum Yes
Rate Amount of Ambient vehicles created for each parking of this type Float Yes

Note: Any parking type not present is the list of ParkingEntry will behave as if it was explicitely defined with a Rate of 0.

<ParkingEntryWeighted>

A parking type with associated number of Parked Aircrafts to spawn per parking of this type on an airport.

Attribute Description Type Required
ParkingType Enum (see the TaxiwayParkings) for a type of parking Enum Yes
Rate Amount of Parked Aircrafts created for each parking of this type Float Yes
GoodToBadMatchRate Factor between 0.0 and 1.0 that determines how much more likely will aircrafts that best match the parking radius be spawned rather than aircrafts with smaller wingspan Float Yes

Note: A GoodToBadMatchRate of 0.0 means only the aircrafts that best match the parking radius will be spawned. 1.0 means any aircraft that fit in the parking is as likely to be spawned. 0.5 means that aircrafts that best match the parking radius are twice as likely to be spawned than other aircrafts.

Note: Any parking type not present is the list of ParkingEntryWeighted will behave as if it was explicitely defined with a Rate of 0.

<LivingWorldRegion>

Corresponds to one of the 26 different regions of the world, and lists various types of vehicles, boats, or human characters, that can be used to populate this region. See the image in this section for correspondances between letter and region.

Attribute Description Type Required
RegionID A single capital letter, ‘A’ to ‘Z’, identifying the current region of the world Enum Yes
RightSideDrive Whether vehicles on the roads drive on the Right side (true), or the Left side (false) Boolean No
Element Description Quantity
<FreewayTraffic> Lists the vehicles to be used for Road Traffic, as well as the relative weight between various models. 0-1
<Airport> Lists the models for various category of airport vehicles and human characters 0-1
<Boats> Lists the boats to be generated, as well as the relative weight between various models. 0-1

<FreewayTraffic>

Lists the vehicles to be used for Road Traffic in this region, as well as the relative weight between various models.

Attribute Description Type Required
TrafficSpeed DEPRECATED. Speed (in km/h) of the vehicles. Not used in MSFS for now. Float No
Element Description Quantity
<ModelEntry> A single model of vehicle, with its associated weight. 1+

<Airport>

Lists various types of vehicles and human characters to be generated in this region. Most vehicle types have their own specificities explained in their respective sections around the GroundVehicles chapter.

Element Description Quantity
<AmbientTraffic> Lists vehicles to be used for Ambient Traffic for this region, wandering accross the airport 0-1
<PushbackTugs> List of Pushback vehicles in this region 0-1
<SmallPushbacks> List of Small Pushback vehicles in this region 0-1
<FuelTrucks> List of Fuel Truck vehicles in this region 0-1
<BaggageLoaders> List of BaggageLoader vehicles in this region 0-1
<BaggageCarts> List of BaggageCart vehicles in this region 0-1
<BaggageDollies> List of BaggageDollies vehicles in this region 0-1
<CateringTrucks> List of Catering Truck vehicles in this region 0-1
<BoardingRamps> List of Boarding Ramp vehicles in this region 0-1
<GroundPowerUnits> List of Ground Power Unit vehicles in this region 0-1
<IdleWorkers> List of human characters to be used as Idle Workers in this region 0-1
<GroundVehiclePilots> List of human characters to be used as vehicle pilots/drivers in this region 0-1
<Marshaller> List of human characters to be used as Marshaller in this region 0-1

<AmbientTraffic>

Lists the vehicle models to be used as Ambient Traffic in airports of this region. See here for more details on how Ambient Traffic works.

Element Description Quantity
<ContainerEntry> A single model of vehicle, with its associated weight. 1+

<PushbackTugs>

Lists the vehicle models to be used as pushbacks for large aircrafts in airports of this region. See here for specificities of those models.

Element Description Quantity
<ContainerEntry> A single model of vehicle, with its associated weight. 1+

<SmallPushbacks>

Lists the vehicle models to be used as pushback for small aircrafts in airports of this region. See here for specificities of those models.

Element Description Quantity
<ContainerEntry> A single model of vehicle, with its associated weight. 1+

<FuelTrucks>

Lists the vehicle models to be used as fuel trucks in airports of this region. See section here for specificities of those models.

Element Description Quantity
<ContainerEntry> A single model of vehicle, with its associated weight. 1+

<BaggageLoaders>

Lists the vehicle models to be used as baggage loaders in airports of this region. See section here for specificities of those models.

Element Description Quantity
<ContainerEntry> A single model of vehicle, with its associated weight. 1+

<BaggageCarts>

Lists the vehicle models to be used as baggage carts/tugs in airports of this region. See section here for specificities of those models.

Element Description Quantity
<ContainerEntry> A single model of vehicle, with its associated weight. 1+

<BaggageDollies>

Lists the vehicle models to be used as baggage dollies in airports of this region. See section here for specificities of those models.

Element Description Quantity
<ContainerEntry> A single model of vehicle, with its associated weight. 1+

<CateringTrucks>

Lists the vehicle models to be used as catering trucks in airports of this region. See section here for specificities of those models.

Element Description Quantity
<ContainerEntry> A single model of vehicle, with its associated weight. 1+

<BoardingRamps>

Lists the vehicle models to be used as boarding ramps in airports of this region. See section here for specificities of those models.

Element Description Quantity
<ContainerEntry> A single model of vehicle, with its associated weight. 1+

<GroundPowerUnits>

Lists the vehicle models to be used as ground power units in airports of this region. See section here for specificities of those models.

Element Description Quantity
<ContainerEntry> A single model of vehicle, with its associated weight. 1+

<IdleWorkers>

Lists the human character models to be used as idle workers in airports of this region. See the Airport Services on how to setup idle worker behaviors.

Element Description Quantity
<ContainerEntry> A single model of human character, with its associated weight. 1+

<GroundVehiclePilots>

Lists the human character models to be used as vehicle drivers in airports of this region. See the pilot section of GroundVehicles and the chapter on Human SimObjects on how to properly seat them.

Element Description Quantity
<ContainerEntry> A single model of human character, with its associated weight. 1+

<Marshaller>

Lists the human character models to be used as marshallers in airports of this region. See the Airport Services and Human SimObjects for more details.

Element Description Quantity
<ContainerEntry> A single model of human character, with its associated weight. 1+

<Boats>

Lists the boat models for this region.

Element Description Quantity
<InlandBoats> List of boat models to be used to populate coasts and lakes 0-1

<InlandBoats>

Lists the boat models to be used to populate coastal areas and lakes, as well as their relative weights.

Element Description Quantity
<ContainerEntry> A single model of boat, with its associated weight. 1+

<ModelEntry>

An entry for a model of SceneryObject to be used in living world, containing its GUID and a weight.

Attribute Description Type Required
ModelID GUID of the SceneryObject to use. GUID Yes
EntryWeight Integer weight to determine relative frequency of this model compared to other (higher weight means higher chances of choosing this model) integer Yes

Note: An EntryWeight might be 0, but the sum of the EntryWeights of a list must be strictly greater than 0 and strictly less than 2147483647

<ContainerEntry>

An entry for a model of SimObject to be used in living world, containing its Title and a weight.

Attribute Description Type Required
ContainerTitle Unique Title identifying the variant of SimObject to use. String Yes
EntryWeight Integer weight to determine relative frequency of this model compared to other (higher weight means higher chances of choosing this model) integer Yes

Note: An EntryWeight might be 0, but the sum of the EntryWeights of a list must be strictly greater than 0 and strictly less than 2147483647
