<RTCSequence>

Note: be careful, an RTCSequence needs to be encapsulated in an RTCSequencer element.

Element Description Type Unique Required Default
<ShotType> Precise the type on which is played the animation Enum Yes Yes undefined
<ShotCount> Number or animation played in the RTCSequence, if not specified or equal to -1 took all the possible animations (max 40 for memory reason) Number Yes No -1
<RootPosition> Position where to place the animation Enum Yes No NoSpecification
<FileToInclude> If present, then, any animation need to come from a gltf file matching any FileToInclude Text No No /
<FileToExclude> If present, then, any animation must not come from a gltf file matching any FileToExclude Text No No /
<AnimationToInclude> Any selected animation for the RTCSequence needs to contain one of the names in any AnimationToInclude Text No No /
<AnimationToExclude> Any selected animation for the RTCSequence must not contain one of the names in any AnimationToInclude Text No No /
<PathToInclude> path to be included in the RTC. Allow to include subfolder that are not includes by default Text No No /
<PathToExclude> path to be excluded in the RTC Text No No /
<DepthOfField> To manage the depth of field (blur amount and focal distance) DepthOfField Yes No See DepthOfField
<FadeFromBlack> If true the RTCSequencer starts with a fade from black Bool Yes No False
<FadeToBlack> If true the RTCSequencer ends with a fade to black Bool Yes No False
<Fov> Fov (zoom) parameters Fov Yes No See Fov
<Shuffle> If we should shuffle the order of RTCShot Bool Yes No True
<CameraLinkedToPlayer> If true the camera of the animation is link to the player Bool Yes No Aircraft: True
Airport: False
POI: False
<TargetLinkedToPlayer> If true the target of the animation is link to the player Bool Yes No Aircraft: True
Airport: False
POI: False

Enum Description
Aircraft For aircraft only
Airport For airports only
POI For POI only

Element Description Type Unique Required Default
<InstanceId> GUID referencing the RTCSequencer GUID Yes Yes /
<Descr> Description, using as a name Text Yes Yes /
<Skippable> If we can skip the animations Bool Yes No False
<Pausable> If we can pause the animations Bool Yes No True
<LoopAnimation> If the RTCSequencer should loop Bool Yes No /

<RootPosition>

<RootPosition> allows to place the RTCSequencer at some specific position, it can be useful to have animations on e.g. a runway or a tower.

Value Description
None No specific position, will be on the plane
DefaultRunway Will be place on the default runway
PlayerAtSequenceStart Will be at the place the plane start
NoSpecification If its airport type it will be as for “DefaultRunway”, if its aircraft type, as for “None”
RunwayStart Start of the default runway
RunwayEnd End of the default runway
RunwayCenter Center of the default runway
Tower First tower of the airport
Parking First parking of the airport
ActivePOI Will be on the active Point Of Interest

<Dof>

Depth of field is a little structure that permits to manage the depth of field in an RTC. By default, the depth of field is activated in the Hangar. You can change the value in real time in the game thanks to the remote tool by activating the tweakable value in the RTC field.

Element Description Type Unique Required Default
<Blur> Blur value ( 0.1 = small amount of blur, 2 = a lot of blur) Float Yes Yes 0.25
<FocaleDist> If present and bigger than 0 then the focal distance will be constant and not auto Float Yes No 0
<Activate> Permit to activate/ deactivate the Death of field, deactivating it permit to have no blur at all Bool Yes No True
<AutoFocusSpeed> Coefficient for the autofocus speed: 0= will not do any focus 1= instant focus Float Yes No 0.05
<ApplyFrontFocus> True : blur depending on the focus dist will be apply even on foreground Bool Yes No False

<Fov>

The fov allows you to modify the actual zoom and to set a delay to arrives to the zoom (to get a little animation of zooming). | Element | Description | Type | Unique | Required | Default | |—|—|—|—|—|—| <FovLength> |Value of the zoom in radian where min is 0.1 (really big zoom) et max is 2.75 (looking from far away) |Float|unique|Optional|0.8| <SmoothTimeFov>|Time to from the old fov value to the given value in seconds.|Float|unique|Optional|0|

<CameraBehaviours>

<CameraBehaviours> contains a set of behaviors, which define how the camera should behave during the RTC.

Property Description Type Unique Required Default
<CameraBehaviour> Describe a behavior CameraBehaviour No No /

<CameraBehaviour>

<CameraBehaviour> is a behavior, which defines how the camera should behave during the RTC.

Element Attribute Description Type Unique Required Default
CameraBehaviourType Type of the behavior Enum Yes No /
<CameraBehaviourAttributes> Attributs link to the type CameraBehaviourAttributes No No /

CameraBehaviourType

Here are for the moment the possible attribute values for CameraBehaviourType: |Value|Description| |—|—| None|No specific behavior | ActorTarget|The camera will follow a given actor; the attribute should be the name of the actor|

<CameraBehaviourAttributes>

CameraBehaviourStrAttribute

Attribute Description Type Unique Required Default
CameraBehaviourStrAttribute Describe a behavior Text Yes No /
