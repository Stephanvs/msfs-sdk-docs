rotor_inside This bus is dedicated to rotor sounds in cockpit view. It has an ambisonic child bus dedicated to Ambisonic takes.

miscellaneous_inside This bus is dedicated to miscellaneous sounds in cockpit view, such as flight control, fuel pumps, hydraulics, and gears. It has an ambisonic child bus dedicated to Ambisonic takes.
rain_inside This bus is dedicated to rain sounds in cockpit view. It has an ambisonic child bus dedicated to Ambisonic takes.

Aux Busses

Game-Defined Auxiliary Sends

In the sound XML, the AcousticParameters contains several ReverberationPreset parameters.

These parameters allows to set up Shareset reverbs into customs auxiliary bus (rev_inside_custom, and rev_outdoor_custom busses) to be used as game-defined busses. So all sounds that are tagged “using game-defined aux sends” in the WwiseProject are sent to the custom bus with the effect shareset you defined in the sound.xml.

That way, every content profider can create their own cockpit and/or outdoor reverberation for their aircraft, by adding the shareset newly created into the wwise package, and setting it up with the right shareset name in the sound.xml. Example:

<Parameter Name="ReverberationPreset" SharesetInside="rev_my_reverb_inside" SharesetOutside="rev_my_reverb_outside"/>

Please have a look at sound.xml categories page to righly set up your acoustic parameters through sound.xml.

Here is the Cockpit Reverb Aux Bus list that are shared in the wwise project. In wwise, aux bus are available following this Master-Mixer Hierarchy path: \Master-Mixer Hierarchy\Default Work Unit\Master Audio Bus\game\game_aux.

Aux Bus Name Description
rev_inside_custom This bus is the one where SharesetInside is setup for player SimObjects.
rev_outdoor_custom This bus is the one where SharesetOutside is setup for player SimObjects.
rev_outdoor_custom_ai This bus is the one where SharesetOutside is setup for AI SimObjects.

User-defined Auxiliary Sends

For outside reverberation, you can also use generic busses we share with the community. All default aircraft models are using these busses with the user-defined mode. Here is the generics Outside Reverb Aux Bus available in the wwise project. Every content provider can use them as user-defined bus.

Aux Bus Name Description
rev_outdoor_aircraft This is generic outside reverb for aircraft.
rev_outdoor_aircraft_low_end This is generic outside reverb with more low end. It is mainly designed for airliners.

Secondary Output Busses

The Master_Headset_Bus , as its name suggests is a secondary Master bus dedicated to sounds heard through headphones using the communication device output bus. This is where you can route avionics warnings that are supposed to play in the user’s headset.

IMPORTANT
These parent busses below only serve to diverge child bus structure. Please, never route any sound through it, otherwise your aircraft won’t benefit from ui volume level mixer, and global dynamic mixing structure.

  • Master_Headset_Bus
  • warnings
Output Bus Outside Description
warning_signal This bus is dedicated to warning signal sounds that come through user headset.
warning_voice This bus is dedicated to voice warnings sounds that come through user headset.
