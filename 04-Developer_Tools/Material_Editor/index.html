Materials

Materials are used to describe how light behave when it hits a surface and is reflected towards the camera. In order to describe this behaviour, our engine needs a Code and a Texture. This combination changes the way the Material is rendered.

Codes for global purposes

  • Material_CODE_DIFFUSE Textures used

    • Diffuse: a texture expressing the albedo, it could be a RGB or RGBA texture.
    • Normal Roughness Metal: Normal in GA (as XZ packed), B: roughness, R: metalness. Please note that the metalness in not apply on the ground.

Codes for visual effects

Editor

Important things to know:

  • Material packages have a strict organization in which:
    • .Material files are in the root directory of the package
    • texture files (.jpg, .jpeg, .png, .bmp, …) should be put in a “Textures” directory

An example of Material package: /SampleMaterials/DefaultDiffuse.Material /SampleMaterials/Textures/Diffuse.png

Main Window

  1. File menu, allows creation of new Materials
  2. Package Selection, you have to select a package in order to modify it’s Material library asset group. You won’t be able to modify a Material without a package.
  3. This is the view of all the Materials available to select. Brown Materials are locked and cannot be modified unless you have the package for. If you don’t have Brown Materials, this means that they are not exposed yet and cannot be used. A package selected loads the Materials within and allows them to be modified. When a Material can be modified it will appear either in grey (not loaded), in red (load failure, error), in yellow (Material currently loading) or white (Material loaded and ready to be modified).

The main window has also some other elements:

  • Filter feature: Allow you to filter the Materials by name.
  • Bitmap Preview option: When enabled, you can see a preview of your Material by simply hovering the mouse over the Materials in the Material list (3.)
  • Tags section: Allow you to organize your work by tagging Materials with several different tags in order to filter them. It is quite handy if you have too much Materials going on. “All” will not filter Materials, “User” will filter Materials you are able to modify, the other tags will be your owns.

Tags

Tags are stored per Material, the editor will fill the tags list based on the Materials he know. The buttons “+” and “-” allows you to add temporary tabs. Temporary, because they won’t be saved until any Material are enlisted with. In order to set a Material to a tag, you have to hit the “Tags” button. This will change the Material list to a checkbox list with the Materials name. Select your tag and then check the Materials you want to enlist with it. A save button should appear.

Known issues

Material Inspector

This is the window that will allow you to rename the Material, change the textures and eventually some parameters.

  1. File menu (“Save” => will save the Material under your asset group. “Delete” => will delete the Material. “Reset” => will reset the default values. “Apply” => will apply the Material to your scene. “Copy Guid” => once the Material is saved we set a guid to it in order to identify, we allow to copy it to your clipboard: this is useful when editing some files by hand. “Exit” => will close the window.
  2. Show, show the texture slots you want. We strongly suggest to refer to the documentation to know whether or not you need to set some textures based on the Code you chose.
  3. Surface Type: This is important only for Materials that will be used in terrain surfaces. It indicated to the engine and the game what kind of surface the Material describes. It will be important on how the simulation behaves as well as the rendering.
  4. Textures display: this window zone will help you set different textures to the Material, just by clicking on the textures. Default display is Black which is no texture at all. In general, you will need at least a Diffuse texture.
  5. The advance features are used for Visual Effects.

Create and Edit Materials

  • First, you will have to create an Asset Group under your working Package with the type MaterialLib. Do not forget to specify a working directory for your Asset Group. Default will be your drive root directory.
  • Then, you will need to Select a package using (2. Main Window)
  • Once you are set, hit the File menu and create a New Material. You should see a new window appear (Inspector Window)
  • You can(have to) name your new Material then
  • You can set the textures you want by clicking on the links in the section 4. (Inspector Window)
  • You can also set the Surface Type, for example: Asphalt for a runway.
  • Then Hit the File menu and Save
  • Then again the File menu and Apply

Edit a Material

  • Be assured that your Material can be edited by checking it’s color on the editor (not Brown!).
  • Then, click twice on the Material name or right click then open on it
  • Then proceed as per Create and Edit Materials

Tags

Tags allow you to organize your work, you can set a Material to several tabs that allow you to filter your Materials with words related to your project like (runways, …)
