Launching the tool

To enable the scenery editor, DevMode must be enabled.

Camera

The developer camera can move freely in the world and focus object. Make sure it is enabled in the menu:

  • To rotate the camera: hold the Alt key, left click and drag the mouse.
  • To move: hold the Alt key, middle click and drag the mouse.
  • To zoom: use the mouse wheel or hold the Alt key, right click and drag the mouse.

Loading

There are several ways to load a scene:

In the scenery editor: select a package and press Load this asset group.

In the project editor: select an asset group with the type BGL and press Load in editor.

Scenery objects

To place a new object in the world:

  1. select the desired object in the list
  2. press Add to add an object in the scene
  3. move and rotate it with a gizmo
    1. Type of object to place (sceneries, runways, polygons…)
    1. List of object that can be added
    1. Button to place the selected object.
    1. List of object in the scene.
    1. The One-click placing mode allows to place the same object several times on the ground by left-clicking. Hold Ctrl key and move the mouse to choose the desired rotation. Hold Shift to control the scale.

Polygons

This tool allows to create and edit polygons in the world. It is possible to attach properties to this surface, which allows to:

  • flatten the terrain;
  • exclude auto-generated buildings;
  • force or exclude vegetation;
  • change vegetation type;
  • specify airport zones (changes the type of auto-generated buildings);
  • apply a material on the ground;
  • add or remove water.

Like a scenery object, press the Add button to place a polygon in the scene. Then hold Ctrl and left-click several times on the ground to define the shape. To finish, double-click or press Enter.

The surface type can be edited in the property box (right-click > properties).

Water Polygons

To add or remove water in an area, edit an existing polygon or create a new polygon and tick the Water box in the properties. Additional properties will appear, allowing you to choose the water type and to define if it excludes water or not by ticking the Exclude box.

Water polygons create a distance field on the water-side of the polygon boundary. This effect can be seen in the shoreline in the image below.

Note: when Exclude is ticked, the water polygon excludes only the type of water that is selected. In the example below, the bigger polygon is OCEAN, so the A-shaped exclusion polygon only works if it is also of type OCEAN.

Water polygons work additively. A regular water polygon increments the “water” value by 1, while a water polygon with Exclude decrements the “water” value by 1. The distance field is applied to this value. A value greater than zero will define water while a value of zero or less will define land.

Rectangles

Rectangles are designed to perform optimized terraforming. But also to exlude buildings and to define airport areas.

A profile can be applied to the rectangle to deform the terrain along a spline.

Landmark location

Allow to place a landmark that will be displayed in the world map. It can be a POI (for example the Eifel tower, the Taj Mahal…), a city or a location with a specific fauna.

The name field is mandatory.

The owner field specifies the creator name of the scenery (should be the same as the field creator in the package)

Airports

The tool allows to edit several airport components:

  • Runways
  • Helipads
  • Taxiways
  • Parkings
  • Taxiway signs
  • Airports surfaces (aprons)
  • Airport lights
  • Painted lines
  • Jetways
  • Towers
  • Windsocks
  • Projected meshes

After selecting them, they can be moved and rotated with a gizmo. Other parameters like surface type, width, or markings need to be edited in the property box.

Runways

Runways can be placed only on an airport.
On Properties of a runway, it is possible to edit the followings parameters:

Configuration

For generic parameters such as heading, length, width, pattern altitude, number and designation.

Materials

To set textures on the surface and display options.

The runway can have several layers of materials. Each one has its own properties:

  • Tiling U: the texture tiling along the width of the runway

  • Tiling V: the texture tiling along the length of the runway

  • Width: ratio of the runway width

  • Falloff: distance in meter of alpha blending at the edges

  • Type: can be one of the following:

    • base tiled: basic tiling along the runway

    • border: apply a texture along the edge of the runway

    • tire gum: This mode split the input texture in 3 and applied it at the start and the end the runway to simulate tire marks on the ground.

Other options are:

  • Transparent: only draw the markings
  • Enable coloration: color settings
  • Ground Merging: merge the texture and the aerial.

Markings

To set different types of markings on runway

Terraforming

To flatten or edit a profil on a runway. The profile editor is the same one as the Rectangle, please refer to its documentation.

Note: The profil can interact with other terraformers (polygons and rectangles) depending of their priorities. The default priority of the runway is 10.

Lights

To add lights in the center and on the edge of the runway.

Pavements

To set additional parameters for runway pavements.

Approach lights

To set the different kind of approach lights.

VASI

To set the Visual Approach Slope Indicator. A runway can have up to 4 VASIs.

Runway start

To set the aircraft position and heading for a start on a runway.

Taxiways

Taxiway paths are segments that link two taxiway points, or a taxiway point and a taxiway parking. A path is moved by moving the points and parkings.

Aprons

They allow to place arbitrary surface on the ground. The edition is the same as for polygons with ctrl + left click.

They have a priority parameter that define their draw order.

Painted lines

Painted lines allow to have custom markings on the ground. Painted hatched area can also be used to draw a polygon with strips inside.

Some type of line can spawn lights on top on them. The edition is the same as for polygons.

Taxiway signs

A taxiway sign is composed of one or several sections. Each section has a type (Location, Direction, Mandatory, Information, Runway, Unknown) and a content. The latter can be composed of any upper case letter (A-Z), any numerical digit (0-9), or a special characters. A tooltip is available to easily remember the allowed caracters :

Jetways

Taxiway parkings with the "Gate" type must have a jetway. They can be composed of several scenery objects or sim objects that link a terminal with a plane.

Projected meshes

It takes a scenery object as child, but instead of drawing it in 3D, the mesh will be projected on the ground and rendered in 2D. It can be useful to avoid z-fighting artifacts with the ground. If several meshes are drawn on top of each other, they must have different priorities. The lowest priority is rendered first. By default, they are drawn below every objects in the airport, this behavior can be changed with the draw before option.

The meshes should be exported without translation, rotation or scale. Theses informations will be ignore by the in-game system.

Only materials that contain these texture combinations can be used: * albedo * albedo + metal roughness AO * albedo + normal map * albedo + normal map + metal roughness AO Note: When they are present, metal and AO components are not used.
